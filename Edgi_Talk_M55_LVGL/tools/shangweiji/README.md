# Edgi Talk 睡姿监测系统 - 上位机项目

## 一、嵌入式端程序简介

### 硬件平台
- **MCU**: Cortex-M55
- **开发环境**: RT-Thread Studio
- **WiFi芯片**: CYW55500 (WiFi 6, 通过SDIO接口, 当前未启用)
- **显示屏**: LVGL GUI

### 数据流架构

```
STM32 (压力传感器阵列)
   │
   │ UART5 @ 115200bps
   │ 268字节帧格式: [0xAA][seq][len=256][ADC×256][CRC8][0x55][填充×6]
   ▼
PSOC M55 开发板
   │
   ├──▶ 压力数据 → 保存至 g_pressure_data[256] (16×16矩阵)
   │       └──▶ LVGL 界面实时显示
   │
    ├──▶ AI 推理 → IMPressure_compute()
    │       │ 输入: 16×16 压力矩阵 (旋转校正)
    │       │ 输出: 7类概率
    │       ▼
    │       [0] unlabeled     未标记
    │       [1] supine        仰卧
    │       [2] right_lateral 右侧卧
    │       [3] right_head    右侧头
    │       [4] left_lateral  左侧卧
    │       [5] left_head     左侧头
    │       [6] noise         噪声
    │
    └──▶ UART2 输出 (调试/上位机数据源)
            ├── 帧解析日志 (ADC十六进制/十进制)
            ├── AI 推理原始输出 (7个概率值)
           └── 睡姿分类结果 (标签 + 置信度 + 累计计数)
```

### 当前通信方式

| 端口 | 方向 | 用途 | 波特率 |
|------|------|------|--------|
| UART5 | RX (输入) | 接收STM32压力数据帧 | 115200 |
| UART2 | TX (输出) | 输出调试信息至串口/上位机 | 115200 |

### 数据帧格式 (UART5接收)

| 偏移 | 长度 | 内容 | 说明 |
|------|------|------|------|
| 0 | 1 | 0xAA | 帧头 |
| 1 | 1 | seq_num | 帧序号 |
| 2-3 | 2 | 0x0100 | 数据长度 (256, 大端) |
| 4-259 | 256 | ADC[] | 16×16压力传感器原始值 |
| 260 | 1 | CRC8 | 校验和 |
| 261 | 1 | 0x55 | 帧尾 |
| 262-267 | 6 | 0x00 | 零填充 |

## 二、上位机开发需求

### Phase 1: UART 串口版 (第一版, 立即开发)

#### 功能需求

**1. 串口数据采集**
- 通过USB串口读取UART2输出
- 波特率115200, 8N1
- 支持自动重连 (串口插拔检测)
- 解析两类关键信息:
  - 压力帧数据 (256字节ADC值)
  - AI推断结果 (睡姿标签 + 置信度)

**2. 实时睡姿显示**
- 当前睡姿状态 (大字显示)
- 7类睡姿概率分布 (柱状图/饼图)
- 睡姿切换时间线 (最近N次切换记录)
- 连续睡眠时长统计

**3. 压力热力图**
- 16×16压力矩阵实时热力图渲染
- 色阶自适应或手动调整
- 刷新率: 跟随设备输出频率 (~1Hz)

**4. 数据记录与存储**
- SQLite本地数据库存储
- 记录内容:
  - 时间戳
  - 睡姿分类
  - 7类概率值
  - 原始压力矩阵 (可选, 用于回溯分析)
- 按"睡眠会话"组织数据 (自动或手动标记开始/结束)

**5. Web界面展示**
- 局域网HTTP服务 (如 `http://192.168.1.100:3000`)
- 手机/电脑浏览器访问
- 页面包含:
  - 实时状态面板
  - 历史数据查询
  - 睡姿占比统计图

#### 技术栈

| 层级 | 技术 |
|------|------|
| 运行时 | Node.js |
| 串口通信 | `serialport` 库 |
| 数据存储 | SQLite (`better-sqlite3`) |
| Web服务 | Express.js |
| 前端界面 | HTML + CSS + JavaScript (轻量级, 无框架) |
| 图表渲染 | Chart.js (浏览器端) |
| 热力图 | Canvas 2D API |

### Phase 2: WiFi 无线版 (后续升级)

#### 改动范围

**嵌入式端 (main.c + wifi_tcp_sender.c)**
- 启用WiFi组件 (`RT_USING_WIFI_HOST_DRIVER`) - 你的项目已启用
- 添加WiFi连接逻辑 (SSID/密码配置) - 你的项目已有 `wifi_gui.c`
- 添加TCP客户端, 连接上位机IP:Port - 已实现 `wifi_tcp_sender.c`
- 将UART2输出同步到TCP - 在 `u2_puts()` 内增加 `wifi_tcp_send_str()` 调用

**上位机端 (server.js + tcp-server.js)**
- 新增TCP Server监听 (如 `0.0.0.0:8888`) - 已实现 `tcp-server.js`
- 保留串口模式作为备选, 可通过配置切换数据源
- 数据解析层完全复用 - `parseLine` 逻辑与串口版一致

**配置方式（嵌入式端）**

修改 `wifi_tcp_sender.h` 中的默认IP/端口：
```c
#define WIFI_TCP_DEFAULT_SERVER_IP    "192.168.1.100"  // 改成你电脑的IP
#define WIFI_TCP_DEFAULT_SERVER_PORT  8888
```

或运行时调用：
```c
wifi_tcp_init("192.168.1.100", 8888);
```

**配置方式（上位机端）**

通过环境变量控制：
```powershell
# 设置Web端口
$env:PORT=3000

# 设置TCP监听端口
$env:TCP_PORT=8888

# 指定串口（多串口时）
$env:SERIAL_PORT="COM3"

# 启动
npm start
```

**工作流程**

1. 板子先连上 WiFi（你已有 UI 输入 SSID/密码）
2. 板子启动后自动尝试连接上位机 `192.168.1.100:8888`
3. 上位机 TCP Server 接受连接，接收到的数据格式与UART2输出一致
4. 上位机解析逻辑、数据库、Web界面与串口版完全相同
- 保留串口模式作为备选, 可通过配置切换数据源
- 数据解析层完全复用

#### 优势
- 无线连接, 无需USB线
- 支持远距离部署 (同一局域网)
- 可多设备同时监听 (TCP多连接)

## 三、关键注意事项

### 数据解析难点
- UART2输出的是**人类可读文本**, 不是结构化二进制
- 需要编写正则/状态机来从文本中提取:
  - 压力数据: `[FRAME #xxx]` 后的256个ADC值
  - AI结果: `=> posture=xxx conf=xx%`
- 建议在嵌入式端后续增加**JSON格式结构化输出**, 简化上位机解析

### 数据存储策略
- 16×16压力矩阵 × 每秒1帧 × 8小时 = ~460KB/夜 (压缩后更小)
- 建议默认只存储睡姿分类结果, 原始压力矩阵按需存储
- SQLite单文件数据库, 易于备份和迁移

### 局域网访问
- 电脑和手机需连接同一WiFi
- 上位机运行后打印访问地址, 如 `http://192.168.1.100:3000`
